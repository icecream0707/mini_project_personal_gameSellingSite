# mini_project_personal_gameSellingSite

### 개요

프로젝트명:  다운받을 수 있는 파일형태의 게임 파일을, 게임이 필요한 사람들에게 판매한다.

개발기간: 그때그때 함. 일단 레이아웃부터. 할수있는 영역에서 최소한의 

기획의도: 상품 판매 사이트 또는 온라인 상품(서비스) 판매 사이트를 구현할 목표가 있다. 첫 프로젝트이므로, 그중에 난이도가 낮은 게임 다운로드 사이트(게임 판매 사이트)를 기획하게 되었다.

### 페르소나 설정

이름: 김민수 (25)

특징 및 행동패턴: 재밌는 게임을 수집하고 체험하는 취미가 있다. 게임을 선택할때 게임의 디스크립션을 보면서 **{게임의 독창성(기발함), 취향이 맞는 게임회사의 게임인지 여부}**를 본다. 그리고 **{때마다의 땡기는 장르}** 하나를 두고 고른다. 대학생이라서 게임 하나에 50000원 이상이면 구매하지 않는 특성이 있다.

페르소나를 통한 규칙 도출(질문과 답변):

**비회원 구매 가능하게 할까? → 불가능하다**. 비회원이 한번 결제해서 다운로드를 단일로 하다가 오류가 생기면, 구매했었다는 증명을 할 수가 없어지고 처리가 난감해진다. 그리고 게임을 선택까지는 가능하게 하되 구매버튼을 눌렀을때에 회원가입 창으로 넘어가게 하여야 회원가입율과 구매율을 높일 수 있다.

**구매될 게임이 빠르게 검색되기 위해서 필요한 기능 →**

1. **“관심 게임회사” 저장 기능**
2. **게임의 장르별 필터링 기능**
3. **가격 필터링 기능**
4. 게임별 디스크립션

### 핵심 데이터 정의

회원 데이터: {유저id:비밀번호}, 유저id가 구매한 게임 리스트, 유저id가 “관심버튼”을 눌렀던 게임회사.

게임 데이터: {게임id : 게임 제목 : 게임회사 : 게임장르 : 게임 썸네일 이미지 : 게임 가격 : 게임 설명 : 게임 다운로드 링크}

### 서비스 데이터 흐름

비회원 흐름(구매 버튼): 게임 찾기 → “특정게임 설명페이지”에서 구매버튼 누르면 “구매 페이지” 대신 “회원가입 + 로그인” 페이지로 넘어감.

비회원 흐름(관심 게임회사 버튼): 게임 찾기 → “관심 게임회사 버튼”을 누르면 “관심 게임회사 페이지” 대신 “회원가입 + 로그인” 페이지로 넘어감.

회원가입/로그인 흐름: 비로그인 상태에서 회원가입을 완료했으면 다시 “회원가입/로그인 페이지”로 돌아옴.

로그인 되어있는 상태에서는, 유저아이디와 로그아웃 기능만 구현한다.

회원가입 and 로그인, 로그인 했을때, 이전 들어가려고 했던 페이지가 있다면, 그 해당 페이지로 이동한다(기억해뒀다가). ( 1/2: 게임 구매 버튼, 2/2: 관심게임회사 버튼 에 구현할 내용 ) 나머지 경우는 사이트 홈페이지로 이동된다.
